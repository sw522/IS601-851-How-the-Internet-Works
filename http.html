HTTP and HTTPS protocols

Pictured Above: The lock on your browser indicated https is being used and your data is secure

     HTTP (Hypertext Transfer protocol) is the language used for communication between web browsers and servers. There are two main types of these communications, the GET and POST requests. The GET request is your browsers way of asking the server for a file (document). When a use visits a web page one of these requests is made to get the file associated with this page which is usually coded in html. The get request will include all the text associated with this file but if there are other objects such as imagines embedded in the code this will trigger another request for that file. These individual subsequent requests will be rendered and displayed as the file arrives. The other type of requests is the POST, which is when you send information to the server such as a username/password during a login or a query using a search engine. If the login scenario the server you are communicating with will also send back some additional data called a cookie. This is a way for a website to identify and remember who you are so that subsequent requests that are sent include that personalized data.

     The Internet itself is an open and shared platform so trouble with HTTP is that since data is sent in plain text it is possible for hackers to intercept communications to spy on you. Safe websites leverage HTTPS in order to protect your privacy. HTTPS leverages SSL (Secure Sockets Layer) or TLS (Transport Layer Security) to create a secure connection between both parties. When a website asks a browser to engage in this connection it provides a digital certificate confirming confirming it is a website it claims to be. These certificates are issues by governing authorities that verify the websites identities like a government issuing passports. If a server tries to initiate a secure connection without a valid certificate the web browser will warn the user that it could not verify its identity.
